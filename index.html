Dodger!

by: Dean Le

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I made this game because I liked the simple idea of dodging lasers and iterating throughout different levels. It also mimicked one the of the games that I made on Scratch, and I wanted to remake it because of that.

2. What I Learned

The one main problem that I learned through user testing was that the player should have smoother movement. The movement of the player was jerky and it could not go diagonally. This had also annoyed me, so I decided to change it to making it more smooth.

3. Architecture of My Game

The GamePanel class is the backbone of the game. It is the class that controls and calls all of the variables and methods that make the game work. It uses event listeners to call different methods in the Player, Laser, and ObjectManager classes. First, the GameObject class is extended from each component of the game, which has an update method, x, y, width, and height variables. Second, each game component that interacts (player and lasers) are constantly updated depending on the actions of the person playing the game. The laser class also has an additional parameter of currentState, to increase the difficulty of the levels as they go on. Then, the ObjectManager class calls these methods and adds the lasers to the frame using a timer. After that, the GamePanel class executes the ObjectManager and Player methods when needed. The currentState variable is increased as the scenes (Menu, AvatarSelect, instructions, etc.) change. Finally, the frame is drawn in the Dodger runner class, and GamePanel is added. It is run by the one and only main method in the project.

4. Technical Implementation

I am proud of the avatar select aspect of the game. This makes the game personable to everyone, and is a strong point of the game. I at first thought that the Player class could access variables in the GamePanel class, but that was not true. Therefore, I wrote a method in the Player class that loads and draws the image at the Player's x and y, depending on the avatarNumber int variable in the methods parameter that is passed when the avatar is selected.

5. Most Challenging Part

The most challenging part of this project was finding the balance between a game too hard and a game too easy. At first, as the levels increased, I decreased the timer's parameter, making ActionPerformed run more frequently, adding more lasers. However, as I found out, changing the timer lags out and can crash the computer. After rebooting, I decided to loop the addLaser() method in the ObjectManager class more as the level increased. However, you cannot access the currentState variable in GamePanel in Object Manager, so I added a method called changeState() that took an int variable which I passed currentState from GamePanel. Another challenge within this one was that I could not beat my own game, to see the winning screen. to get passed this, I made a activateInvincibility() method that made the Player's collisionBox outside the frame.

6. Future Fixes and Updates

I would like to add more levels, and powerups that decrease the amount of lasers for a set amount of time. This would make it more engaging and interactive with the player.
